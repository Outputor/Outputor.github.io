入坑Markdown

发表于 2018-01-16 | 分类于 随笔
字数统计: 3.4k | 阅读时长 ≈ 12


自从去年迷上Markdown这种神奇的写作方式，就给我打开了一扇新的大门，从此之后便开始经历了各种Markdown编辑器，最后选择了Markeditor作为我的主力编辑器已经有小半年了，它是一款国产良心软件，剁手120入的，只能用惊艳来形容。附下载链接

    让编辑器更好用

  • markdown只有黑色的字体，好在markeditor还提供了字体样式的选择，我们可以调用HTML的语言让它的格式丰富一些，最让我惊喜的就是可以改变字体的颜色，这对于做笔记来说更多了一个强调的样式，也使文章更加多彩。
    使用方法如下
    <font face="楷体" size=5 color="red">这里面的字体是红色的5号楷体</font>
    如果这样写的话就是下面的效果这里面的字体是红色的5号楷体
  • 列表与制图
    Markeditor采用的是作者自己改造的表格，相对来说更加好用，感觉也更加漂亮，基本的操作已经够用了， 如果比较专业的表格和图表肯定是比较吃力的。

以上是我练习打字的速度折线图，纯粹用Markeditor生成的，很漂亮有木有。
具体操作看说明书。

  • 公式插入
    说到公式插入，作为一个工科生肯定是比较头痛的事，因为貌似我们除了word之处找不到别的可以用来写公式的软件，word里面的公式是图片格式，一不小心就乱了，这是比较恶心的；当然还有一个比较专业的软件LaTex，那个学习成本就比较高了。
    那么markdown就可以很简单的解决这个问题，具体操作看说明书。
    介绍一下如何进行快速的公式输入，其实很简单，不必去死记硬背那些数学符号的表达，在线公式编辑器是帮你把实在的公式转化成你所需要的符号的一个网站，有点类似于word的公式编辑器,你在上面输入，下面就可以直接展示是什么公式，比较适合我这种小白，然后再把文本框里的东西复制拷出来就行了。


以上是这个公式的书写过程
有一个问题需要说明一下，就是Markeditor里没有说明如何将公式与文字放在一起排版，它的说明书里只说明了如何用公式，并且用那种方法生成的公式单独占了一行，如果有将公式放在一段文字里的需求的话就不会了，比较尴尬。
其实非常简单，
$公式$这样写的话就会和文字在一起了，
$$公式$$这样写的话公式就会单独占一行,
$A^3$ 后面还可以接着写东西，这是最大的不同，而如果用$$就不行了。
$$A^3$$

  • PDF分页也是非常有用的，比如我们在写完一个章节之后要写下一个章节就要进入下页了，这时用一下[PAGE]就可以了。
  • 图片插入的解决方案
    图片在Markdown里面必须用图片网页的形式，厉害的是Markeditor提供了直接将图片复制粘贴到里面来的方法，虽然可行，但是，有缺点，一个是图片会跟着一起复制到你所写的文档之下，如果图片删了了可能档里的图片就显示不出来了；另一个就是当转成微信的富文本的时候图片会缺失。
    如果嫌麻烦的人直接把图片拖到里面就行了， 如果不嫌麻烦可以花上几分钟注册个图床，再下个软件的话那用起来的感觉是非常爽的。
    我用的七牛图床，就是在其官网上注册一个七牛云账户，其目的是你的图片上传到其云上。(PS:这可不是祃百度云)

    注册账号之后再注册一个存储空间，然后找到图中密钥部分，还有在新建空间界面找到测试域名，这就是后面要用到的内容。
    然后再下载一个神器MPic，其目的就是快速生成你所需要的图片Markdown格式。

    将其对应填入就ok了。
    这是MPic的官方教程，说得还是非常详细的。
    具体用法就是将图片拖入软件里面，就会自己上传到云，再生成一个Markdown图片格式，直接Ctrl+V就可以了。屡试不爽。
    最后，如果感觉到好用的话请打赏作者。
  • 首行缩进
  • 中文输入法支持
  • 全文检索
  • 软件置顶
  • 单个文件独立窗口
    以上这四个功能是我认为它与其它软件不同的地方，也是吸引我的地方(当然，还有很多别的吸引我的东西)。
    首行缩进应该是中文写作的特色，这也只有本土化软件才能做的如此细致；中文输入法的支持就是在打一些中括号或者是书名号的时候不用切换输入法到英文状态，就可以直接识别转换成英语符号，这个也爽到飞起；全文检索是指在文件菜单栏下的全文检索，而不是Ctrl+F的那个检索，有点Everything的感脚；软件置顶是在某些特殊需求下将软件放在最前面，QQ这个不要脸的天天这么干。

    使用过程中遇到的问题

  • 跨设备同步问题
    这也是个老生常谈了，要求无非就是速度快，安全。我选择的解决方案是坚果云，把文件夹丢里面就不用管了，因为Markeditor就是基于文件夹树的一个管理方案，同步性极其好，基本上在电脑上做了修改，在另一台终端上立刻就能同步，更令人惊喜的是坚果云的手机客户端竟然可以打开.md文件，这样如果我真的有需要的话，也可以在手机上临时查看和修改我的文件了。
  • 标题同时加大，加粗，居中
    我是这样解决的，应该还有更好的办法，基本方法就是调用Html里面的语法。如果以后遇到此类问题也可以照猫画虎，如法炮制。
    <center><font size=6><b>浅析王小波《黄金时代》语言特色</b></font></center>

  • 图片大小调整
    这个就比较简单了，说明里有，一共有两种办法，见说明里的基本用法。
  • 打印页面格式设置
    因为转出来的pdf格式是不能设置的，个人感觉Markeditor导出的pdf页边距太大了，简直不能忍，我的解决方案是导成HTML的格式，然后再用IE进行打印，这时就可以设置页眉页脚以及页边距了，据我所试，火狐和Chrom貌似在打印这块都不行。
  • 微信输出
    如果你有一个微信公众号，那么Markeditor就更值得你拥有了， 微信自己的写公众号的网页简直不能忍，Markeditor自带导出为微信公众号格式的选项，直接Ctrl+V在里面就行了，基本不用改动(当然，只是说文字和图片比较多而没有那么多复杂格式如图表之类的内容)。也是令人怦然心动的一个功能。
  • 特殊符号
    这里的特殊符号有两类，一种是指用于写Markdown标记语法的符号在文本中以普通符号来表示，如->#…，其使用方法是在其前面加上一个反斜杠，就可以当作普通符号来使用了。

另外一种符号就是比较一些特殊的字符， 比如 θ δ π ∀ © 等等。
